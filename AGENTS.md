# 托业错题知识沉淀 · 工作指令

这是一个托业（TOEIC）错题处理仓库。用户丢进来一道做错或想收藏的题，你把其中可复用的知识点提取出来，归类沉淀进 `notes/` 下的笔记，供日后复习。

## 触发条件

仅当用户**粘贴了一道托业题**（题干、选项，文字或截图均可，可能附正确答案）时，才执行下面的写入工作流。

没粘题时不要主动改任何文件——用户可能只是提问、讨论，或让你查阅已有笔记，此时正常回答即可。

## 工作流

**1. 先读后写**
动手前先读 `notes/` 里相关文件，掌握已有条目和它们的格式。新增沿用该文件既有格式；某个词 / 搭配 / 语法点已经记过，就在原条目上补例句或补角度，不要另起一条重复。

**2. 拆题**
定位考点（考词义、搭配、时态、从句、指代……）→ 正确项为何对 → 每个干扰项为何错，逐个说清。题干给出中文译文（填空题代入正确答案译整句），帮助看懂题意；选项是词 / 短语时不逐个翻译。听力题（Part 1–4）额外补：连读 / 弱读等语音现象、题干与音频的同义替换、场景信号词。

**3. 归类**
把拆出来的知识点放进对应文件（见下方分工）。一道题常同时涉及多个维度，可同时写入多个文件——比如一道考 `be responsible for` 的题，可同时进固定搭配、高频词汇和错题本。

**4. 写入**
`notes/` 下所有笔记都使用追加方式：新增错题、新增词汇、新增搭配、新增语法点等一律加在对应文件最后，保持前面已有条目的顺序。已存在的词 / 搭配 / 语法点仍在原条目补例句或补角度，避免重复建条。用户给出的选项频率不写入笔记；用户选错了哪个选项也不写入笔记。

**5. 重生成查看器**
写入笔记后运行 `python3 build.py`（或 `python build.py`）重新生成 `index.html`，把双击版同步到最新。没有 Python 跑不了就跳过，并在回报里提示 `index.html` 待刷新。

**6. 回报**
用 3–5 行说明更新了哪些文件、各自加了什么。不要复述原题。

## 文件分工

| 文件 | 装什么 |
|------|--------|
| `notes/01_错题本.md` | 完整题目存档 + 解析，其他笔记的索引源 |
| `notes/02_高频词汇.md` | 单词的词性、释义、题中例句、易混词提示 |
| `notes/03_固定搭配.md` | 动词与介词搭配、固定短语 |
| `notes/04_语法点.md` | Part 5/6 语法考点，规则 + 例证 |
| `notes/05_长难句.md` | Part 7 长难句的结构拆解 + 翻译 |
| `notes/06_商务场景表达.md` | 邮件 / 会议 / 合同等商务场景表达 |
| `notes/07_易混词辨析.md` | 形近、义近词辨析 |

分类不够用时，新增文件、编号顺延（08、09……），并在 `README.md` 文件表里登记。

## 写作约束

- 中文讲解；单词、例句保留英文原文。
- 例句一律取自题目本身，不自己另造。
- 解析讲清「为什么」对、「为什么」错，不要把选项翻译一遍。
- 选项只保留原始选项文本，不保留频率 / 比例；答案处只写正确答案，不记录用户选错的选项。
- 直给结论。禁用「让我们」「总而言之」「值得注意的是」这类套话。
- 不堆叠加粗，不套多层列表，能一句话说清就别拆三层。
